# 时间管理大师 MVP 规则

## 任务池

系统管理未来 30 天任务。每个任务至少包含：

- 任务标题
- 所属项目
- 四象限分类
- 重要性：S/A/B/C
- 紧急性：high/medium/low
- 截止时间
- 状态
- 下一步动作
- 完成标准
- 预计耗时
- 拖延次数
- 卡点

## 每日调度

每天 08:30 只锁定 1-3 个关键任务。

优先级规则：

1. 重要且紧急优先。
2. 截止时间越近越优先。
3. S/A 级任务优先。
4. 被拖延过的任务加权提高，避免一直逃避。
5. 不重要不紧急默认不进入今日主计划。

## 拖延干预

第一版先在规则层记录，不做自动多轮催促：

1. 到点提醒当前任务。
2. 15 分钟未回复，标记一次拖延。
3. 用户回来后，只推进当前这一件。
4. 任务过大时降级成 15 分钟版本。

## 飞书对话格式

新增任务：

```text
新增任务：7月8日前确认直播助教和销售跟进分工
```

生成今日任务：

```text
今日任务
```

任务完成：

```text
完成：直播 PPT 大纲
```

卡住：

```text
卡住：直播邀约话术，不知道怎么写不显得太功利
```

复盘：

```text
复盘：今天完成了 PPT，话术没发给销售，原因是没有确认销售名单
```

后续版本再对这些命令做完整状态更新。
