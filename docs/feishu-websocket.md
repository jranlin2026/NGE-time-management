# 飞书群消息长连接接入

目标：你在飞书群里发消息，时间管理大师自动接收并处理。

推荐用飞书 WebSocket 长连接模式，不需要公网 URL，也不需要内网穿透。

## 已实现能力

项目已经新增：

```powershell
npm.cmd run feishu:listen
```

监听进程收到飞书群消息后会处理：

```text
今日任务
```

立刻重新生成今天最多 3 个关键任务，并下发到飞书群和飞书任务清单。

```text
新增：7月8日前确认直播海报
```

把新任务写入知识库任务池。是否进入今天的 1-3 个关键任务，由时间管理大师判断。

```text
帮助
```

查看群内可用指令。

```text
完成：直播海报确认
```

反馈已完成，时间管理大师会提醒你继续推进下一个关键任务。

```text
卡住：直播演练 不知道怎么开口
```

反馈卡点，时间管理大师会把任务降级成 15 分钟最小推进动作。

```text
我今天状态不好，不想做直播演练
```

这会被判断为拖延/逃避信号。时间管理大师不会把它当闲聊，也不会新增任务，而是直接拉回当前最高优先级任务，并给出 15 分钟动作。

```text
我有点烦
```

非任务闲聊不会进入任务池。时间管理大师会提醒你改成「新增」「卡住」「完成」或「今日任务」，避免用聊天替代行动。

## 飞书开放平台配置

1. 打开飞书开放平台，进入当前企业自建应用。
2. 进入「应用能力」，启用「机器人」能力，并设置机器人名称。
3. 重新发布应用。
4. 把这个企业自建应用机器人添加到要管理任务的飞书群里。
5. 进入「事件订阅」。
6. 启用「使用长连接接收事件」或「WebSocket 长连接」。
7. 添加事件：

```text
im.message.receive_v1
```

8. 如果飞书提示权限不足，给应用开通消息接收相关权限，并重新发布应用。

注意：飞书「自定义群机器人 webhook」只能发送消息，不能接收群成员消息。接收群消息必须用「企业自建应用机器人」。

## 必开权限

在「权限管理」里至少添加：

```text
im:message.group_at_msg:readonly
```

用途：接收群聊中 @ 机器人的消息。

建议同时添加：

```text
im:message.p2p_msg:readonly
```

用途：接收用户发给机器人的单聊消息。

如果希望群里不 @ 机器人也能被接收，再申请：

```text
im:message.group_msg
```

用途：读取群组中所有消息。这个权限更敏感，只建议用于个人时间管理专用群。

每次添加权限后，都要到「版本管理与发布」重新发布应用。

## 本地启动监听

在项目目录运行：

```powershell
cd D:\CODEX项目\N哥的时间管理大师
npm.cmd run feishu:listen
```

看到类似输出代表监听已经启动：

```text
Feishu WebSocket connected. Waiting for group messages...
```

这个窗口需要保持打开。只要它开着，飞书群消息就会进入时间管理大师。

## 测试方式

1. 启动监听：

```powershell
npm.cmd run feishu:listen
```

2. 在飞书群里发送：

```text
@时间管理大师 帮助
```

3. 如果群里收到「时间管理大师已在线」，说明链路打通。第一次测试建议 @ 机器人。

4. 再发送：

```text
今日任务
```

应该会重新下发今日关键任务，并创建飞书任务清单。

## 注意

- 这不是 Windows 任务计划，它是飞书事件监听进程。
- 监听进程关闭后，群消息不会进入本地项目。
- 每天 8:30 自动下发任务仍由 Codex 自动调度负责。
