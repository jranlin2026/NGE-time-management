# PRD：个人飞书时间管理教练

## 背景与目标

增长负责人每天被临时会议与消息打断，无法判断当前最重要的事，导致个人 IP 和极享 OS 长期不推进。系统每天 08:00 在个人专用飞书群下发当天作战计划，以明确时间段、最小下一步、群内反馈和 10+10 分钟催办，帮助用户从“等完美再做”转向“先完成，再优化”。

## 用户与权限

唯一使用者为增长负责人。飞书群仅本人可见；系统可读写该群消息、@ 提醒、发送卡片、记录完成/卡住/推迟原因。任务来源为知识库与群内消息。

## 核心流程

1. 08:00 从 SQLite 任务池与知识库导入待办，生成当天最多 5 个关键任务。
2. 个人 IP 默认优先，至少 2 项；极享 OS 至少 2 项；只在极享 OS 出现影响使用的 Bug 时提高其优先级。
3. 个人 IP 排入 10:00–12:00、14:00–16:00；极享 OS 排入 10:00–12:00、14:00–24:00；12:00–14:00 不排任务。每个关键任务不超过 120 分钟。
4. 每个关键任务带 1–5 个可勾选关卡。群内点“开始、卡住、推迟、完成”后更新状态和下一步。
5. 到任务开始后 10 分钟未反馈，群内 @ 提醒；再过 10 分钟未反馈，发送教练式提醒并要求选择“开始 15 分钟、说明卡点、推迟并说明原因”。
6. 临时事项仅在用户明确“必须本人参加才能解决”时打断当前计划；被打断任务立即重排。
7. 24:00 前输出 10 分钟复盘，记录完成、推迟原因、拖延次数与明日候选，并写回知识库。

## 数据与业务规则

| 数据 | 规则 |
| --- | --- |
| `tasks` | 记录项目、预计时长、下一步、完成标准、关卡 JSON、完成状态与推迟原因。 |
| `schedule_blocks` | 保留计划版本；单日最多 5 个不同任务。 |
| `task_events` | 记录开始、完成、卡住、推迟、第二次未响应、临时打断。 |
| `manager_settings` | 保存 5 项上限、项目最低数量、项目时间窗、08:00/24:00 节点与催办间隔。 |

## 验收标准

- 给定 6 个以上待办，08:00 生成的计划至多含 5 个不同任务，且个人 IP 和极享 OS 各至少 2 项（池中任务充足时）。
- 给定个人 IP 与极享 OS 同时高优，个人 IP 先被排入可用时段；标记“影响使用的 Bug”的极享 OS 可反超。
- 给定 12:00–14:00，计划不生成任务块；个人 IP 块只落在其允许时段；单个任务块不超过 120 分钟。
- 给定任务开始后 10 分钟和 20 分钟都未反馈，群内依次产生 @ 提醒与含推迟原因入口的教练式催办。
- 给定完成、卡住或带原因推迟操作，任务状态、事件、群内卡片和后续计划一致更新。
- 给定当天结束，24:00 生成复盘 Markdown 到知识库，并包含完成率、推迟原因、拖延次数和明日候选。

## 非目标

- 不做多人协作、团队权限或会议日程自动读取。
- 不替用户判断“是否必须本人参会”；仅根据其显式反馈执行。
- 不把飞书群消息以外的敏感内容自动同步到群内。

## 一周业务验收

- 发布 10 条个人 IP 短视频。
- 极享 OS 上线线索、客户、订单、提成、售后 5 个功能。
- 每天至少完成 3/5 个关键任务，并在 24:00 前完成复盘。
